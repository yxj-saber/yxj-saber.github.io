1. 命名关键字参数

函数定义时中间加上*

*前面的参数叫做位置参数，传参时对号入座即可

*后面的参数叫做命名关键字参数，传参时必须“参数名=参数值”的形式传递

2. Redis简介

  • KV存储系统 – REmote DIctionary Server
  • 提供高速缓存服务 – 缓存热点数据（访问量大，数据量不大）
  • 缓解了数据的压力（高频访问数据不用直接访问数据库）
  • Redis提供了两种持久化数据方案
    1. RDB - 默认开启
    2. AOF – 默认关闭（appendonly yes, 开启降低性能，更能防止数据丢失；关闭相反）

3. Redis安装及简单操作

  • 安装Redis

    wget http://download.redis.io/releases/redis-5.0.3.tar.gz

    gunzip redis-5.0.3.tar.gz

    tar -zvf redis-5.0.3.tar

    make && make install

  • linux作业相关指令

    jobs – 显示当前作业

    bg %[作业号] – 让作业号对应的作业在后台运行

    fg %[作业号] – 让作业号对应的作业在前台运行

    启动作业命令后加 & – 让指定作业启动并放到后台运行

    快捷键ctrl + z – 将正在前台运行的作业放到后台运行

    快捷键ctrl + c – 终止正在前台运行的作业

  • Redis基本操作

    redis-cli -h [主机地址] -p [port] -a [password] – 连接指定主机的指定端口并验证身份

    redis-cli – 连接本机Redis服务器

    auth [password] – 验证身份

    ping - 心跳事件（验证服务器连接）

    set [key] [value] – 放键值对

    get key - 取值

    expire time – 设置超时时间（单位：s）

    del key – 删除键

    keys * – 查看所有键值对

    exists key – 判断key是否存在

    flushdb – 清空当前数据库中的所有键值对

    flushall – 清空所有数据库中的所有键值对

    select （0~15）– 切换底层数据库（默认16个数据库）

    dbsize – 查看当前数据库共有多少组键值对

    save / basave – 保存数据 / 后台保存数据

    shutdown – 关闭服务器

    quit – 退出
