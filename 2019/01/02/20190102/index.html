1. 终端创建python项目虚拟环境和Django项目

  • python项目虚拟环境：

    配置python和pip全局变量

    安装虚拟环境：pip install virtualenv

    创建虚拟环境文件夹：virtualenv –no-site-packages -p C:\python36\python.exe name

    • –no-site-packages –> 创建干净的环境，不包括全局的任何包
    • -p poision –> 指定python版本，并给出python的路径
    • name –> 创建虚拟环境的文件名字

    查看安装的库：pip list

    安装指定的库：pip install xxx

    激活虚拟环境：cd name/Scripts 执行activate激活虚拟环境

    ​ 执行deactivate退出虚拟环境

  • 创建Django项目：

    cd E:thirdprograph/worksapce

    django-admin startproject day01

2. Django模式

  • MVT设计模式

MVT设计模式

​ M全拼为Model，负责和数据库交互，进行数据处理。

​ V全拼为View，接收请求，进行业务处理，返回应答。

​ T全拼为Template，负责封装构造要返回的html。

  • MVC设计模式 – 所有框架遵循的模式

MVC设计模式

​ M全拼为Model，主要封装对数据库层的访问，对数据库中的数据进行增、删、改、查操作。

​ V全拼为View，用于封装结果，生成页面展示的html内容。

​ C全拼为Controller，用于接收请求，处理业务逻辑，与Model和View交互，返回结果。

3. Model简单应用

  • 创建tb_student的表

    (1) models.py中创建名为Student的class类，实为给数据库创建一张名为tb_student的表

    (2) 在setting.py中的INSTALLED_APPS中添加django名字

    (3) 控制台执行python manage.py makemigrations创建迁移配置文件0001_initial.py

    (4) 控制台执行python manage.py migrate将配置文件的内容迁移到数据库

    • 数据库会创建名为tb_student的数据表
